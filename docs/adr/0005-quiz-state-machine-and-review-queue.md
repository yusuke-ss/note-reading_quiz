# 0005. クイズ進行を useReducer の純粋な状態機械 + FIFO 復習キューで実装する

## Status

Accepted (2026-07-08)

## Context

クイズは「10問1セット、即時正誤フィードバック、間違えた音はセット末尾で再出題、ただし復習の結果はスコアに数えない」という、状態遷移とビジネスルールが絡み合う進行ロジックを持つ。UIの都合(タイマー、アニメーション)からロジックを分離し、Vitestで網羅的にテストできる形にしたい。

## Decision

- クイズ進行は `src/lib/quiz.ts` の `quizReducer`(`useReducer` 用の純粋関数)として実装する。状態は `{ queue, current, phase('question'|'feedback'|'finished'), lastAnswer, results, startedAt }`。
- 出題は `generateQuestions()` でレベルの白鍵プールから10問をランダム抽出する。制約は「直前の問題と同じ音は出さない」「同じ音は10問中最大2回まで」。
- 不正解時、`current.isReview` が false であれば復習アイテム(`{ note, isReview: true }`)をキュー末尾に追加する。ただし同じ音がすでに復習キューに入っていれば追加しない(**同一音の復習は最大1回**という不変条件)。
- 復習アイテムの解答結果は `results` に含めない(`current.isReview` が true の場合は `results` を更新しない)ため、スコアは常に最初の10問分のみで決まる。
- 進捗表示は `getProgress()` が導出し、通常時は `n / 10`、復習中は `10 / 10(復習 +m)` という文字列を返す。
- 正解後・不正解後の自動送り時間は `getFeedbackDelayMs(correct)` が返す(正解0.7秒/不正解1.8秒)。この値もUIから独立した純関数として切り出している。

## Consequences

- 状態遷移とビジネスルール(復習キューのFIFO性、スコア計算対象)が純粋関数にまとまっているため、Vitestで「復習は最大1回」「スコアは最初の10問のみ」等の不変条件を直接テストできる(`src/lib/quiz.test.ts`)。
- UI層(`QuizScreen`)はディスパッチとタイマー設定に専念でき、進行ロジックの詳細を知らなくてよい。
- 復習キューは「末尾に追加」の単純なFIFOであるため、例えば「間違えた直後にもう一度出す」といった別の復習戦略に変えたい場合は `quizReducer` 内の1箇所を変更すれば済む設計になっている。
