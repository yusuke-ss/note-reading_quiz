# 0006. 判定はオクターブ厳密・白鍵のみ・音名はドレミ表記とする

## Status

Accepted (2026-07-08)

## Context

このアプリの学習目的は「五線譜上の音符の位置」と「鍵盤上の物理的な位置」を1対1で結び付けて覚えることにある。ピアノ初心者とその家族(日本語話者)を想定ユーザーとしている。

## Decision

- 音の同一性判定は `NoteId`(例: `"C4"`)の完全一致で行う。`letter` が同じでも `octave` が異なれば不正解として扱う(オクターブ厳密判定)。
- 出題・鍵盤生成は白鍵のみ(`whiteKeysInRange`)を対象とし、臨時記号(シャープ/フラット)は出題しない。ただし鍵盤UI自体には黒鍵も表示し、タップは可能(`keyIdToMidi` がシャープ付きIDにも対応)だが、黒鍵を押した場合は常に不正解になる。
- 音名表示は西洋式(C, D, E...)ではなく日本のピアノ教育で一般的なドレミ表記(`SOLFEGE`: ド・レ・ミ・ファ・ソ・ラ・シ)を用いる。鍵盤上のドレミラベルはユーザー設定でON/OFF可能。
- 中央ド(C4)には鍵盤上に常時マーカー(●)を表示し、オクターブ位置の基準点を視覚的に提供する。

## Consequences

- オクターブ違いを不正解として扱うことで、「同じ音名だが違う高さ」を曖昧に許容してしまう学習上の抜け穴を防いでいる。
- 黒鍵を表示しつつ常に不正解にする設計により、鍵盤全体の見た目は本物のピアノに近いまま保ちつつ、出題対象を白鍵のみに単純化できている。
- ドレミ表記は日本語UI・日本のピアノ教育慣行に合わせた選択であり、英語圏の音名表記(ABC式)を望むユーザーには別途設定が必要になるが、現状はスコープ外としている。
