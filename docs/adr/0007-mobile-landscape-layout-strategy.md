# 0007. モバイル横向き対応のレイアウト戦略

## Status

Accepted (2026-07-08)

## Context

主な利用シーンはスマートフォン・タブレットで、特に横向き(landscape)で鍵盤を大きく表示したいという要件がある。横向きでは画面の縦幅(viewport height)が非常に狭くなり、ヘッダー・譜面・鍵盤・フィードバック表示のすべてを同時に確保しようとすると、譜面や鍵盤が潰れてしまう問題が繰り返し発生した(git log の d75bf9e〜99d01a0 で段階的に対処)。

## Decision

以下4つの対策を組み合わせる(`src/screens/QuizScreen.css`, `src/components/StaffDisplay.tsx`)。

1. **CSS Grid の `minmax` 行で譜面エリアに最低高を保証する**: `.quiz-screen` は `grid-template-rows: auto minmax(120px, 1fr) minmax(160px, min(28vh, 220px))`(縦向き)/ `minmax(170px, 1fr) minmax(100px, min(38vh, 280px))`(横向き)とし、譜面行は「余った分だけ伸びるがこれ以下には縮まない」最低高を持つ。鍵盤側は自身の `minmax` の下限まで先に譲る。
2. **StaffDisplayはレベル別の固定 viewBox クロップを使う**: VexFlow描画後のSVGに対して、レベルごとに事前計測した固定の `y` 範囲(`LEVEL_Y_RANGE`)でviewBoxを設定し、`object-fit: contain` で拡大する。毎回 `getBBox()` を取って音符ごとにクロップする方式は、問題によって音符サイズが変動してしまい(加線の本数が変わるため)採用しなかった。固定テーブル方式ならレベル内でスケールが安定し、なおかつレベル間で無駄な余白をクロップできる。
3. **横向きではヘッダーを絶対配置オーバーレイにする**: `.quiz-header` を横向き時に `position: absolute` にして譜面行の上に重ね、ヘッダーがGrid行を1つ消費する分(約44px+gap)を譜面エリアに明け渡す。
4. **FeedbackBanner も絶対配置オーバーレイにする**: 正誤フィードバックのバナーは高さを消費するプレースホルダーとして配置すると譜面の高さを圧迫するため、絶対配置にして通常レイアウトの高さ計算から除外する。

## Consequences

- 横向きでも譜面が視認できる最低限の高さを常に確保しつつ、鍵盤側はスペースがあるときだけ大きく表示される優先順位になっている。
- 固定viewBoxクロップ方式は「その端末・そのレベルでの見え方」を事前計測に依存するため、VexFlowのバージョンアップ等でレンダリング結果が変わった場合は `LEVEL_Y_RANGE` の再計測が必要になる(コメントに計測手順が残されている)。
- ヘッダー/バナーの絶対配置化は、位置決めの基準(`position: relative` な祖先要素)を明示的に管理する必要があり、`.quiz-staff-area` がその役割を担う。
- この設計変更は5つのコミット(d75bf9e, 8929e56, d8c4080, 3c71a9c, 99d01a0)にわたる段階的な調整の結果であり、各コミットメッセージがそれぞれの問題と対処を記録している。
